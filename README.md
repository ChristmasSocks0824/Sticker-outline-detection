# 嵌入式電腦視覺技術之手機殼貼紙輪廓檢測
(Embedded Computer Vision Technology for Mobile Phone Case Sticker Outline Detection)

## 1. 專題簡介
本專題旨在將電腦視覺演算法推向邊緣運算（Edge Computing），透過 Android Studio 開發一款具備「極簡使用者介面」的行動軟體，將手機模擬為工業相機，進行生產線上客製化手機殼的「貼紙輪廓檢測」。

* **解決的問題與應用情境：** 
  坊間許多初學者在學習 AI 或 OpenCV 時常感到抽象難懂。本系統不僅可作為工業上輔助瑕疵檢測（Defect Detection）的原型，更能用最直觀的互動體驗，引導青少年與初學者輕鬆跨入電腦視覺的世界。
* **目標使用者：** 
  對 AI/電腦視覺感興趣的初學者、學生，以及未來可能應用於產線上的檢測操作員。
* **系統價值：** 
  提供低成本、高便攜性的邊緣運算檢測方案，並具備高度的教育與展示價值。

## 2. 系統需求
* **輸入來源：** 實體相機（由上往下俯視擷取畫面）。
* **輸出結果：** Android App 介面即時顯示處理後的「貼紙輪廓檢測」結果。
* **環境限制：** 待測手機殼需放置於**黑色背景**上以利特徵擷取。
* **即時性需求：** 只需要靜態影像即可
* **成功條件：** 在教室內有日光燈(光源均勻的環境)做拍攝取得輸入影像，能呈現完整目標物件的真實色調即可以辨識成功。

## 3. 硬體與軟體環境
* **硬體平台：** 
  * Sony 智慧型手機 (Xperia系列)
  * 實驗工作平台（包含垂直俯視視角之相機鏡頭支架、黑色背景板）
* **軟體與開發工具：** 
  * Android Studio
* **電腦視覺技術/函式庫：** 
  * OpenCV 3.10：負責核心的影像前處理與輪廓偵測。

## 4. 安裝方式
1. 下載本專案程式碼：`git clone https://github.com/ChristmasSocks0824/Sticker-outline-detection`
2. 建議安裝 Android Studio 2025.3.4版本。
3. SDK套件安裝: compileSdk: 35、minSdk: 24 (Android 7.0 以上)
4. Python 環境：專案會透過 Chaquopy 自動配置Python 3.10，您不需要在手機上手動安裝 Python。
5. 連接 Android 實體手機或模擬器進行編譯。

## 5. 執行方式
1. 將待測物（手機殼）放置於工作平台的黑色背景上。
2. 開啟 App，並允許相機權限，App使用者介面上有設置「Open Camera」，點擊按鈕開啟相機。
3. 開啟相機後，將手機鏡頭對準待測物，確保畫面為垂直俯視。
4. App使用者介面上有設置「Capture 」拍攝按鈕，點擊按鈕擷取當前影像。
5. 擷取當前影像後，介面上有「mode」選擇模式的選單點選任一個mode才能做「影像辨識」的處理。
6. 選擇其中一個mode後，介面上有「Detect」做辨識的按鈕，點擊一下後做影像處理，等待處理結果。
7. App使用者介面上有「擷取原始影像」+「辨識結果影像」的顯示視窗，點擊視窗可以放大/縮小觀察影像。
8. 點擊一下顯示視窗，就會跳出去回到原本App使用者介面，即可以重複執行以上1~7步驟使用App功能。

## 7. 模型或演算法說明
本系統核心影像處理流程如下：
1. **影像擷取：** 透過手機相機取得即時畫面，介面設置拍照按鈕，點擊按鈕擷取當前影像送往Python後端。
2. **影像解析度校正：** Java端先根據實體相機的感光元件角度旋轉影像縮放尺寸1024*1024(Pixel)
3. **裁切影像的尺寸：** Python端負責影像處理，輸入影像是採用「中心正方形裁切」縮放尺寸512*512(Pixel)
4. **前處理：** 對輸入影像轉HSV拆分通道對V亮度通道做做Gamma矯正 + CLAHE => 矯正後根據「目標物件的色調」來建置專用的遮罩(例如:目標物件有黃褐色與白色組成的貼紙，可以手動調整HSV遮罩的色調範圍)。
5. **特徵擷取：** 第四個步驟會得到乾淨的二值化遮罩，使用bitwise_and將原彩色輸入影像與二值化遮罩合併，產生黑色背景貼上原彩影像 => 做Canny + dilate 根據影像的中位數動態精準繪製出「目標物件封閉輪廓」。
6. **輪廓檢測：** 第五個步驟會得到乾淨的目標物件輪廓的影像，設計一套自適應面積篩選機制，剔除雜訊+繪製通過篩選的目標物件輪廓，把每一個物件依序登記編號與數據(辨識成功的輪廓+面積數據)做儲存資料。
7. **圖像切割** 第六個步驟有儲存資料，使用enumerate功能，App操作介面上可輸入物件編號取得對應的輪廓資料(例如:輸入編號4)，把把編號4輪廓用cv2.FILLED填滿封閉輪廓，接著用步驟4.的方法在全黑色背景影像上貼上編號4的彩色圖像。

### 📸 情境測試與環境抗性說明

本次測試針對不同背景、光線與拍攝角度進行了實測，以評估系統在各種現實條件下的邊界抓取與辨識能力：

1. **理想條件（專屬相機支架拍攝）** 🏆
   * **結果**：最佳。系統可以正確辨識到手機殼上的**每一張貼紙**，並準確分配編號，無雜訊干擾。

2. **黑色桌面場景測試**
   * **正面拍攝**：容易產生嚴重反光，進而影響並干擾周邊白色區塊的辨識。
   * **手機後傾 35 度**：由於距離與視角改變，導致邊界抓取發生錯誤（例如：同一張貼紙被錯誤分割成兩個獨立的物件）。
   * **後傾 20 度 + 順時針轉 30 度**：後傾角度減少後，圖形抓取的完整度有所上升；但依然受到反光影響，偶爾會發生圖形漏抓的狀況。

3. **白色桌面場景測試**
   * **正面拍攝**：受反光影響較小，但會因為光線產生「陰影雜訊」，使得系統將陰影誤判為輪廓邊界。
   * **手機後傾 20 度**：傾斜後，光線造成的陰影雜訊明顯增加（畫面周圍出現錯誤輪廓），但圖片主體的抓取完整度比正面拍攝時還要高。
   * **後傾 20 度 + 順時針轉 30 度**：增加旋轉角度後，周圍的陰影雜訊有減少的趨勢，但會導致部分圖形抓取失真（例如貼紙輪廓變形）。

**總結**：
系統在光源穩定、角度垂直的狀態下表現極佳；但將其應用於一般桌面時，演算法容易受到「深色桌面反光」與「淺色桌面陰影」的干擾。未來可能需要針對光影變化加入去雜訊（Denoising）或邊緣強化處理，以提升不同角度下的容錯率。
